Problem

The existing dashboard was two pages of OECD score tables. HDFC could look up a number but couldn't answer the questions that drive programme decisions: which of the 10 implementing partners is underperforming? Which intervention is working? How does the programme track across India's states? After 1.5 years of engagement, this had never been built. Samarth took over as SPOC in March 2026 — the wireframe was accepted within the same quarter.

What I built
5-section wireframe — Impact Overview, Parameter Scoring (year-over-year), OECD Analysis, State-wise India map, Project Performance across partner / regional / intervention views
Data logic sheet — all programme indicators mapped to database schema tables
Reckoner automation — twice-monthly partner reports moved from fully manual to automated via Python + Power BI, with a delivery interface built into the dashboard itself
Research before design

Before touching the wireframe, attended HDFC's national programme dissemination — sat through presentations from all 10 NGO implementing partners, took notes on what they flagged as broken, and cross-questioned for clarity. The regional comparison view, partner accountability section, and intervention performance ranking in the final wireframe came directly from what partners raised on that floor.

⚡

Built using Claude CLI throughout — information architecture, scripting, logic building, and data mapping. Claude Design Tool for prototyping and wireframe iterations. Claude Power BI extension to identify what data transformations were needed at each step. Multiple client review cycles with revisions; wireframe formally accepted by HDFC.
